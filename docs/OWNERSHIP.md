# Ownership and naming

This project is currently published under `Akaro96/codex-cursor-bridge`. That is normal for a small solo-maintained open-source developer tool.

## Current setup

- GitHub repository: `Akaro96/codex-cursor-bridge`
- Maintainer: `Akaro96`
- Planned npm package name, if published later: `@codex-cursor-bridge/cli`

The npm package is intentionally not published yet. Source installation from GitHub is the supported path for v0.1.0.

## When a personal account is fine

Publishing under a personal GitHub account is common for solo-maintained open-source projects, especially prototypes, developer tools, and early releases. It is simple and transparent: users can see who maintains the project.

## When an organization might make sense later

If the project grows beyond a small personal-maintainer tool, a neutral GitHub organization such as `codex-cursor-bridge` could help with:

1. neutral project branding,
2. adding co-maintainers,
3. matching a future npm scope,
4. separating project ownership from a personal profile.

A transfer is optional. If it happens later, update repository badges, security advisory URL, CODEOWNERS, npm metadata, package scope, and release checklist in the same change.
